class Node:

    def __init__(self, cargo=None, next=None):
        self.__cargo = cargo
        self.__next  = next

    def value(self):
        return self.__cargo

    def __str__(self):
        return str(self.value())

    def print_list(self):
        print(self, end="")   # print my head
        tail = self.__next    # go to my next node
        if tail is not None : # as long as end of list was not reached
            tail.print_list() # recursively print remainder of list

    #...s

Un peu de théorie et de vocabulaire¶

Qu'est ce qu'une liste chainée ?¶

C'est une structure de données qui implémente une collection d'éléments sous la forme d'une séquence de noeuds connectés.

Qu'est ce qu'une structure de donnée ? Donnez un exemple.¶

C'est un moyen de faciliter l'organisation et le regroupement de donner pour faciliter leur utilisation. Une liste de liens est un exemple typique de structure de données. Elle permet de regrouper des éléments en les organisant dans une structure que l'on peut parcourir de manière séquentielle.

Expliquez la différence entre un noeud (Node) et sa cargaison (cargo).¶

Un noeud est un élément d'une liste chainée. Il s'agit d'un objet qui contient une référence intégrée à un autre objet du même type ET un élément de données appelé sa cargaison. La cargaison est composée des données contenues par un noeud. La référence est ce qui permet de relier les noeuds entre eux !

Expliquez la différence entre un noeud (Node) et une liste chainée (LinkedList).¶

Le noeud sert à créer la liste chainée. Donc, une liste chainée est un ensemble de noeuds enchaînés. Il existe plusieurs types de liste chainée : doublement chainées, simplement chainées, cyclique, ...

Dans le livre et le cours magistral, la notion d'une liste chaînée a d'abord été expliqué sans la classe LinkedList, simplement en chaînant des noeuds l'un à l'autre. Pourquoi avons nous alors besoin d'une classe LinkedList ?¶

  1. la classe LinkedList nous permet de représenter de façon plus cohérente des listes vide.
    Question : avec uniquement la classe Node, comment serait représenter une liste vide ?

La liste serait représenter par le type None, pas d'entreprise, pas de wagon, pas de chauffeur, pas de gare, pas de locomotive, nada, nothing niks, rien, get it ? Or avec une classe LinkedList, elle serait bien du type LinkedList, vous auriez la sncb mais sans aucun équipement.

  1. la classe LinkedList est la classe qui représente la structure de données qu'est une liste chaînée. La linked list contient la structure et les données de la chaîne, ainsi que l'ensemble de méthodes définies sur cette structure de données.

    Pour faire le parallèle, la sncb(liste) représente le réseau belge mais ses trains(noeuds) ne le représente pas entièrement. La sncb contient les trains, les marchandises à l'intérieur (pas de trafic humain svp, soyons sérieux), les différents cadres(structure) et travailleurs(méthodes) pour faire marcher la société.

Remarque : en pratique vous pourriez cacher l'implémentation de Node dans LinkedList.

Question 8¶

Imbriquer une classe dans une autre¶

En Python il est possible d'imbriquer une classe dans une autre. Ceci peut être utile si une classe n'est utilise que par une seule autre classe. C'est le cas ici avec la classe Node. Ce n'est qu'une classe auxiliaire pour la classe LinkedList. Adapter le code de votre classe LinkedList en imbriquant le code de la classe Node dedans. Il suffit d'indenter le code de la classe Node et de changer chaque référence à la classe Node à l'interieur de la classe LinkedList par self.Node.

  • Il suffit d'indenter la classe Node dans la LinkedList et remplacer des appels par self.Node.

Quels sont les avantages et désavantages d'imbriquer la classe Node dans la classe LinkedList?¶

  • Avantage : la classe est cachée de l'extérieur de la classe LinkedList.
  • Désavantage : la classe Node devient plus difficile à tester indépendamment, code de la LinkedList devient long et la classe Node n'est plus utilisable pour créer d'autre structure.
