class Node:

    def __init__(self, cargo=None, next=None):
        self.__cargo = cargo
        self.__next  = next

    def value(self):
        return self.__cargo

    def __str__(self):
        return str(self.value())

    def print_list(self):
        print(self, end="")   # print my head
        tail = self.__next    # go to my next node
        if tail is not None : # as long as end of list was not reached
            tail.print_list() # recursively print remainder of list

    #...s

Qu'est ce qu'une liste chainée ?¶

C'est une structure de donnes qui implmente une collection d'lments sous la forme d'une squence de noeuds connects.

Qu'est ce qu'une structure de donnée ? Donnez un exemple.¶

C'est un moyen de faciliter l'organisation et le regroupement de donner pour faciliter leur utilisation. Une liste de liens est un exemple typique de structure de donnes. Elle permet de regrouper des lments en les organisant dans une structure que l'on peut parcourir de manire squentielle.

Expliquez la différence entre un noeud (Node) et sa cargaison (cargo).¶

Un noeud est un lment d'une liste chaine. Il s'agit d'un objet qui contient une rfrence intgre un autre objet du mme type ET un lment de donnes appel sa cargaison. La cargaison est compose des donnes contenues par un noeud. La rfrence est ce qui permet de relier les noeuds entre eux !

Expliquez la différence entre un noeud (Node) et une liste chainée (LinkedList).¶

Le noeud sert crer la liste chaine. Donc, une liste chaine est un ensemble de noeuds enchans. Il existe plusieurs types de liste chaine : doublement chaines, simplement chaines, cyclique, ...

Dans le livre et le cours magistral, la notion d'une liste chaînée a d'abord été expliqué sans la classe LinkedList, simplement en chaînant des noeuds l'un à l'autre. Pourquoi avons nous alors besoin d'une classe LinkedList ?¶

  1. la classe LinkedList nous permet de reprsenter de faon plus cohrente des listes vide.
    Question : avec uniquement la classe Node, comment serait reprsenter une liste vide ?

La liste serait reprsenter par le type None, pas d'entreprise, pas de wagon, pas de chauffeur, pas de gare, pas de locomotive, nada, nothing niks, rien, get it ? Or avec une classe LinkedList, elle serait bien du type LinkedList, vous auriez la sncb mais sans aucun quipement.

  1. la classe LinkedList est la classe qui reprsente la structure de donnes qu'est une liste chane. La linked list contient la structure et les donnes de la chane, ainsi que l'ensemble de mthodes dfinies sur cette structure de donnes.

    Pour faire le parallle, la sncb(liste) reprsente le rseau belge mais ses trains(noeuds) ne le reprsente pas entirement. La sncb contient les trains, les marchandises l'intrieur (pas de trafic humain svp, soyons srieux), les diffrents cadres(structure) et travailleurs(mthodes) pour faire marcher la socit.

Remarque : en pratique vous pourriez cacher l'implmentation de Node dans LinkedList.

Question 8¶

Imbriquer une classe dans une autre¶

En Python il est possible d'imbriquer une classe dans une autre. Ceci peut tre utile si une classe n'est utilise que par une seule autre classe. C'est le cas ici avec la classe Node. Ce n'est qu'une classe auxiliaire pour la classe LinkedList. Adapter le code de votre classe LinkedList en imbriquant le code de la classe Node dedans. Il suffit d'indenter le code de la classe Node et de changer chaque rfrence la classe Node l'interieur de la classe LinkedList par self.Node.

  • Il suffit d'indenter la classe Node dans la LinkedList et remplacer des appels par self.Node.

Quels sont les avantages et désavantages d'imbriquer la classe Node dans la classe LinkedList?¶

  • Avantage : la classe est cache de l'extrieur de la classe LinkedList.
  • Dsavantage : la classe Node devient plus difficile tester indpendamment, code de la LinkedList devient long et la classe Node n'est plus utilisable pour crer d'autre structure.
